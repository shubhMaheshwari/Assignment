1 Introduction

This lecture is a quick introduction to some basic elements of Javascript.

We will look at how to define functions, objects and methods in Javascript and how to work with some basic built-in functions in Javascript.

Javascript's syntax is somewhat like that of C. In addition, Javascript has constructs that might be familiar to you from your experience with C or similar languages: while loops, assignment, return statements, etc. But we will not focus on these in this lecture. Javascript exceptions (try catch) will be covered in a later lecture.

Ctrl-Shift-J

3 Literal objects in JS

We begin with literal objects: 

var a = {x: 3, y: 4, greeting: "Hi"};
var b = {}; // empty object

var declares a variable (in this case at the top level) and optionally allows an initialization.

4 Field access

4.1 Field access and update

console.assert(a.x === 3);
a.x = 7;
console.assert(a.x === 7);

You may wish to compare this with the field access we did in Racket earlier: (a 'x) and (a 'x 7).

4.2 Asserts

The assert statement is like the check functions in Racket. A true assertion returns nothing; a false assertion raises an "Assertion failed" error.

4.3 Equality

Note the equality symbol has three '' signs. There is also equality with two '' signs, whose behaviour is sometimes surprising. We will mostly use the three = sign for equality.

4.4 More general syntax for fields

The following additional syntax is also supported for field access: 

a['x'] = 10;

The object may be treated as a vector indexed by an expression evaluating to a symbol.

5 Functions in Javascript

var f = function (x, y) {
        return x+y;
};

The function construct is similar to Racket's lambda. Notice the C like syntax, however.

The body of the function consists of statements terminating with a semi colon. The return keyword is required, like in C.

6 Functions are objects too

Every function in Javascript is also an object. You can freely attach fields to a function: 

f.xyz = 10;

7 Defining and installing Methods

7.1 Defining methods

A method is a function that operates on an object. 

var dist = function () {
        return this.x + this.y;
}

Note the special identifier this, which is an implicit first parameter to the function.

7.2 A syntactic variant

function dist() {
        return this.x + this.y;
}

7.3 Naming conventions

Method names start with a small letter and use camel case like manhattanDistance.

7.4 Installing methods

To installing the method dist object a, we write 

a.dist= dist;

We could have used any other name on the left hand side. For example: 

a.anyname = dist;

7.5 Method call

Recall that when an object's method is called the following happen:

  1. The method name is used to locate the function in the object.
  2. The located function is then called with the object as the first parameter.
console.assert(a.dist() === 14);

7.6 The apply method

Each function object has a built-in method called apply. This method is used to apply the function on an object and any additional arguments that the function may take. 

console.assert(dist.apply(a), 14);

Calling a method a.dist() is equivalent to applying the function against dist to a. 

console.assert(a.dist() === a.dist.apply(a));

We have seen a similar situation in our Object implementation in Racket where we had (mcall a 'f) equivalent to ((a 'f) a). The only difference is that while application of a function to an object argument requires no special syntax in Racket, in JS, to apply a function to an argument, we need to use the apply method of the function.

8 Constructors

8.1 What is a constructor?

A constructor in Javascript is a function that is used to populate the fields of an existing object.

8.2 Example

var Point = function (x,y) {
    this.x = x;
        this.y = y;
};

8.3 Convention for naming Constructors

A constructor is named with a capital letter. Compound words use 'camelcase': e.g., DataManager.

You may want to read about JS coding conventions here:

  • Javascript Naming Conventions
  • Douglas Crockford's JS Coding conventions

9 Constructing objects

Objects are created using the new construct: 

var pt = new Point(2,3);

9.1 Constructors of literal objects

Even literal objects have a constructor, which confusingly enough is the function object called Object.

9.2 The constructor field

Every object has a constructor field which is automatically initialized to the constructor of the object. 

console.assert(pt.constructor === Point);

10 Lexical scope in Javascript

10.1 this

  • this is a lexical identifier.
  • It is not declared but bound to an object in a constructor or function.

10.2 The top level

When Javascript is run in the browser, the top level environment is available as the window object. At the top level the identifier this is bound to the window object.

When run in a standalone interpreter like node, the identifier this at top level is bound to a ``global'' object that is not a window.

10.3 var declarations

  • A top level var declaration creates a binding in the window object.
  • A var binding within a function creates a binding limited to the function body.
  • A var binding within a block (enclosed in { and }) has scope within that block.

10.4 Functions built using function are methods

In the object system we designed in Racket, functions were fundamental; methods were functions whose first argument was an object.

In Javascript, when we define a function using the function keyword, we are defining a method.

10.5 bind

Given a function f and object a, f.bind(b) returns a function. This function, when applied to an object a, ignores the object and instead, applies f to b.

The bind function comes in handy when we write event handlers (discussed later).

10.5.1 Example

a = {x: 2, y: 3};
b = {x: 1, y: 6};
a.dist = dist;
console.assert(a.dist() === 5);
console.assert(a.dist.bind(b)() === 7);
console.assert(a.dist.apply(b) === 7);

10.5.2 Understanding bind

Here is a simple model in Racket for bind. bind simply takes a method m and an object o and returns a new method. The new method ignores its first parameter (ignore-obj) and applies the method m to o ad any additional arguments. 

(define a (obj '((x . 2) (y . 3))))
(define b (obj '((x . 1) (y . 6))))
(define dist
  (lambda (this)
    (+ (this 'x) (this 'y))))

(dist a) ; => 5
(dist b) ; => 7
(a 'dist dist) ; install dist against name 'dist
(mcall a 'dist) ; => 5

(define bind
  (lambda (m o)
    (lambda (ignore-obj . args)
      (apply m (cons o args)))))

((bind dist b) a) ; => 17

Of course, in Javascript, bind is itself a method installed on the object m, so there is an additional level of indirection.

So, in effect, we could have had something like the following, if we had implemented functions as objects in the Racket model. Modifying our simple Racket model, is possible and not too hard to do, but that would take us too far. 

;;; this won't run, because in our Racket  model
;;; functions are not objects.
((mcall (mcall a 'dist) 'bind b))
;;; compare with  a . dist . bind (b)()

11 Prototype inheritance

11.1 The prototype field

  • Each function object comes pre-equipped with a field called prototype.
  • The prototype field is initially an empty object.

11.2 Inheritance via the constructor's prototype

  • When a field f is looked up in an object, the object's own fields are searched first.
  • If the object doesn't have that field, then the object's constructor's prototype is searched to locate f.
  • Thus any field added to the prototype field is accessible to any object built by the constructor.

11.3 Example and the hasOwnProperty field

// pt was earlier declared as 
// pt = new Point(2,3)

console.assert(pt.constructor === Point);

// add field z to Point's prototype object
Point.prototype.z = 10;

// pt inherits the field z
console.assert(pt.z === 10);

// pt doesn't own z; it inherits it
console.assert(pt.hasOwnProperty('z') === false);
console.assert(Point.prototype.hasOwnProperty('z') === true);

12 Modules in Javascript

In this section, we describe how to make and use javascript modules.

12.1 Export required objects

First we install the objects that we want exported in the exports object: 

exports.Point = Point;
exports.dist = dist;

12.2 Requiring the module

Assume that our module is stored in index.js.

We import the module using the require line. 

var basic = require("./index.js");

All exported objects are now available as fields to the object basic created as a result of a require invocation. (Type the rest of the lines in this example in the js shell.) 

console.assert(basic.dist.apply({x:3, y:4}) === 7);
var p2 = new basic.Point(1,2);
console.assert(p2.x === 1);

The object p2 continues to inherit from Point's prototype: 

console.assert(p2.z === 10);

13 Running javascript code in the browser

The javascript code running in the node interpreter needs to be bundled together to be able to run in a browser.

13.1 Bundling

This assumes you have the npm webpack Converting the index.js module to a browswer-compatible bundle.js module (you could choose any other name) is done using a simple command line, which we wrap into a makefile 

pwd=$(shell pwd)

all:
	npm run webpack ${pwd}/index.js ${pwd}/bundle.js

clean:
	\rm -rf *.html *.js

and run it from the command line prompt 

make -k

13.2 Creating HTML Boilerplate code

The boilerplate code in an html file (say code.html) includes a reference to bundle.js: 

<!doctype html>
<html>
  <head>
    <title>Code: Objects in JS</title>
  </head>
  <body>
    <script src="bundle.js"></script>
  </body>
</html>

13.3 Testing for errors

Once the file is loaded into the browser, you may want to switch to the console (Ctrl-Shift-j).

Any errors in your program will show up in the Javascript console.
