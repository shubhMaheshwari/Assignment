3.2 literate-tools: About, Setup & Usage

3.2.1 About

literate-tools is a repository which contains a makefile and supporting files to help us build literate programs. We will use it by running a file called init.sh. which brings in literate-tools and sets up the makefile that lets us build our code and documentation from org files.

3.2.2 Setup

  1. Video Instructions
    • Follow this link (click here)
  2. Text Instructions

    Make sure your directory structure is as follows: 

    -------------------------
    <roll-no>
     |
     `----src
    	  |
     	   `----labs
          		|
          		`--- lab-3
             		  |--- index.org
------------------------
    1. Go to <roll-number> folder.
    2. Download init.sh from this link (Click here) and save it at <roll-no>/init.sh
    3. Run $ chmod +x init.sh && ./init.sh
    4. Ignore make errors that you recieve while running this: $ pushd .; cd ~/emacs/lisp/org-8.2.10; make; popd; = This navigates to =~/emacs/lisp/org-8.2.10 and then builds org-mode, which is the package that powers our literate tools
    5. $ echo "Hello world" >> labs/lab-3/index.org.
    6. At <roll-number> folder, run $ make

3.2.3 Usage

Every time a change is made to one of your .org files, run make from <roll-number> folder. This will recreate all your built files in <roll-number>/build/docs and <roll-number>/build/code.

3.3 Customising Emacs

Things you can do:

  • change fonts, font families, and sizes
  • keybindings
  • set variables like proxy
  • activate or deactivate modes:
    • ido
    • rainbow-delimiters-mode
    • blink-cursor-mode

3.3.1 elisp in the *scratch* buffer

  • switch to the scratch buffer. To run elisp code in the *scratch* buffer, navigate to the end of this expression, and execute using C-j.
  • activate ido-mode, with:
(ido-mode t)
  • set blink-cursor-mode to nil:
(blink-cursor-mode t)
  • set http-proxy, using:
(setq url-proxy-services
   '(("no_proxy" . "^\\(localhost\\|10.*\\)")
     ("http" . "proxy.iiit.ac.in:8080")
     ("https" . "proxy.iiit.ac.in:8080")))

3.3.2 .emacs

.emacs is an elisp file that runs on Emacs initialization. Copy the above changes into your .emacs file

3.4 Terminal in Emacs

This link is a good overview of the various options for running shells inside Emacs.

Two modes offer shell in Emacs: M-x shell and M-x term.

  • M-x shell is just a wrapper around your shell program, as specified by the variable explicit-shell-file-name or the SHELL environment variable.
  • M-x term is a fully featured terminal emulator. For example, one can run vim, top, etc. inside the term, which is not possible inside shell. On the other hand, shell is more widely portable, so will work over SSH (though in this situation, expecting vim is much more realistic than expecting emacs + term). shell is also available inside windows.
