      • Undo

3 Problems

3.1 Emacs

3.1.1 Files and Keys

Find the key sequence that performs the following actions:

  • Open a new file named hello.txt
  • Enter Info, the documentation browser.
  • Enter the Emacs manual in Info.
  • Copy the first line of the Emacs manual into hello.txt 5 times.

3.1.2 Navigation and Commands:

Find the command sequence that moves around a buffer as specified.

  • Up 5 lines, right 4 chars, down 3 lines.
  • Up 1 screen, right 3 words, left 5 chars.

3.1.3 Help

Answer the questions below, and list the sequence of actions you took to find the answers in Emacs.

  • What command is C-x C-c tied to?
    • What does that command do, according to the documentation?
  • What does the command dired do?
    • What keys is it bound to?

3.1.4 Custom

Run the command customize-face, hit enter (RET is what docs call the Enter key, by the way), type default, then RET again. Perform the actions listed, and write out what you did to make them happen. (Be brief, please.)

  • Change the font size to 100.
  • Change the foreground color.
  • Change the font.

3.1.5 .emacs preview

Open the file ~/.emacs. Save its contents to you org file.

3.1.6 Bonus

Talk to the doctor for a while. Save the conversation!

3.2 Org-mode

3.2.1 Document structure

Make a file called main.org.

  • Title it Org-mode Lab, using the #+TITLE header.
  • Create #+AUTHOR and #+EMAIL headers.
  • Make a heading called Org 1.

3.2.2 Markup and Plain Lists

  • Inside it, make a plain list.
  • Make a series of items in the plain list, with bold, italic, code, and verbatim formats respectively.

3.2.3 Source blocks

  • Make a Racket source block. Inside it, put the ring-area function you created in the Intro to Racket lab.
    • Provide this function using provide.
  • Make a Racket source block. Inside it, write a program that will output a list of the first 20 even numbers.
    • Provide this function using provide.
  • Bonus: Find out how to pass arguments to code blocks, and create a code block for a Hello World program that says, "Hello, " followed by your name, in C. Pass your name to it as an argument.

3.2.4 Links

Make a heading called Links. Create a plain list with items according to the instructions below.

  • Make a link to your Markups and Plain Lists heading.
  • Make a link to this lab page on the course portal.
  • Make a link to hello.txt (that you created using Files and Keys).

Hint: Use Hyperlinks help from Org mode (inside Info in Emacs, or online).

3.2.5 Bonus:

  • Make a target inside the Org 1 heading, and link to it. Title it Target Link.
  • Store a link to the org Info page, and make a link using that stored link.
  • Store a link to the Python source block in Source Blocks.
  • Store a link to the table in Bonus: Tables.

3.2.6 Bonus: Tables

  • Make a 3 column table.
  • Title the columns A, B, A ∧ B. (Hint: Use \land for ∧).
  • Write the truth table for A ∧ B.

3.2.7 Export

After finishing all the exercises, export to html.

  • using C-c C-e h or M-x org-export-as-html if org version >8
  • using C-c C-e h h or M-x org-html-export-as-html if org version 8+

4 Instructions

  • Write out and save the above problem solutions - to both the Emacs and Org-mode sections - in a single file called main.org. Follow the guidelines for format.
  • Export to main.html once the entire exercise is done.
  • Commit main.org, main.html, hello.txt.

In your Gitlab directory, you must follow the directory structure as below: 

     -------------------------
     <roll-no>
     |
     `----labs
          |
          `--- lab-2
               |--- main.org
               |--- hello.txt
     ------------------------

6 Guidelines

  • Every solution should be under a heading with the problem title.
  • Keybindings should be printed code, eg. C-x C-f. (Use ==).
  • Commands should be code, eg. =find-file. (Use ==).
